Valorant Esports Platform

A full-stack web application for managing Valorant esports scrim, team rosters, player statistics, and live match results — built from scratch as a personal capstone project.

Problem Statement

Esports teams often struggle to find balanced scrim opponents, manage training schedules, track player performance, and organize team strategies efficiently. The lack of an integrated platform leads to fragmented information, communication issues, and less effective team development.

This project set out to develop a platform tailored to the unique needs of esports teams, bringing scrim matchmaking, performance tracking, strategy management, and team communication into a single, integrated experience.

Objectives

  • Design and implement a relational database schema that models teams, players, matches, and per-map statistics.
  • Build a RESTful API layer with proper authentication, input validation, and error handling.
  • Create a front end that presents live and historical match data in a readable, professional format.
  • Implement WebSocket-based live chat functionality for real-time communication.
  • Achieve full CRUD operations on all major entities through an admin dashboard.

System Architecture

The platform is built using a combination of PHP and MySQL for core application logic and data management, while Node.js and Socket.IO are used to support real-time chat functionality. Integration with the Riot Games API enables the system to access and display up-to-date Valorant player statistics, providing teams with valuable performance insights.

Database Design Overview

The schema was designed around six primary entities with deliberate normalisation to avoid data duplication while keeping query complexity manageable.

  • users — stores user account information, including username, email address, encrypted password, profile image, account role, and registration details. This entity serves as the foundation for authentication and user management throughout the platform.
  • teams — contains team-related information such as team name, logo, description, region, average MMR, and team captain details. It also manages team membership relationships and team identity within the platform.
  • scrim_requests — manages scrim matchmaking requests between teams, including requested MMR range, preferred practice schedule, request status, and participating team references. This entity supports the process of finding suitable scrim opponents with comparable skill levels.
  • schedules — stores training session information, including practice dates, times, descriptions, participating teams, and notification settings. It helps teams coordinate training activities and maintain organized practice schedules.
  • player_stats — stores player performance data retrieved from the Riot Games API, including Rank, MMR, K/D ratio, ACS, match history, and other competitive statistics. These records are used to monitor player development and evaluate training effectiveness.
  • strategies_and_lineups — maintains team strategies, tactical notes, and agent lineup information for different maps. Team members can create, update, and share strategic resources, allowing knowledge and gameplay plans to be stored in a centralized repository.

Key Features

  • Player Profiles - Create and manage player profiles with stats and achievements
  • Team Management - Create teams, manage members, and handle invitations
  • Career Matching - LFP (Looking for Players) and LFT (Looking for Team) system
  • Leaderboard - Track player rankings and VALORANT Premier statistics
  • Scrim Management - Organize and manage scrimmage matches
  • Strategy Planning - Create and share game lineups and strategies per map
  • Team Analytics - Analyze team and player performance metrics
  • Real-time Communication - Team chat with WebSocket support
  • Notifications - Real-time notifications system (Socket.IO)
  • Admin Dashboard - User and team management, ban system
  • Email Support - PHPMailer integration for notifications and password resets

Challenges Encountered

  • Real-time communication architecture
    One of the main challenges was implementing a real-time chat system. During development, it became clear that a traditional PHP-based approach was not suitable for real-time communication because updates would require frequent page refreshes or polling mechanisms. To overcome this limitation, a dedicated Node.js server with Socket.IO was introduced, enabling instant message delivery and a significantly smoother user experience.
  • Interactive strategy planning interface
    Developing the strategy planning feature required working with the HTML Canvas API for the first time. This involved creating an interactive environment where users could place, move, and organize tactical elements on a map, similar to planning tools used in competitive Valorant gameplay. Learning how to manage canvas rendering and user interactions was a valuable technical challenge throughout the project.
  • Project planning and database evolution
    As this was the first large-scale project developed independently, estimating system requirements and future feature expansion proved challenging. Several new requirements emerged during development, leading to multiple revisions of the database schema. Although these changes increased development time, they highlighted the importance of database planning, scalability considerations, and iterative system design in larger software projects.

Lessons Learned

  • Planning the database schema thoroughly before writing a single API route saved enormous refactoring time.
  • Real-time features add significant complexity — they should be treated as a separate layer, not bolted on.
  • Role-based access control is easiest to implement as middleware that reads from JWT claims rather than per-route checks.
  • Component architecture in React benefits from a clear data-flow direction; mixing server state and local state leads to bugs fast.
  • Postman collections as living API documentation are invaluable — both for personal reference and for sharing with future collaborators.
